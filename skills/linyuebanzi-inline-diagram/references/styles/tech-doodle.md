# 风格: 技术简笔画风 (tech-doodle)

**适用场景**: 技术博客、知识管理、架构讲解、个人品牌内容。传达"极简、专业、Geek 审美"的感觉。

## 风格基因(每个提示词开头必须包含)

```
Style: Hand-drawn tech doodle, fine-liner ink pen sketches with organic, slightly imperfect variable-width black strokes.
Coloring: Minimalist felt-tip marker style using highly desaturated, semi-transparent pastel tones (very pale desaturated orange and extremely faint pastel blue). Background is a solid, warm off-white cream color (#FDF9F4).
Character: A tiny, simple, light-gray outline minimalist stick figure character in the bottom-right corner, with a single-line hand-drawn oval thought bubble above its head.
Bottom Highlight: A prominent summary golden sentence at the very bottom, in bold handwritten Chinese text, with a horizontal soft pastel-yellow translucent highlighter marker stroke underneath to make the text pop.
Overall: Clean typography, flat 2D vector, perfectly balanced negative space, professional geek aesthetic.
Do not use photographs, 3D rendering, glossy effects, heavy shadows, dark backgrounds, or dense BI dashboard visuals.
```

## 补充说明

- **主笔**: 黑色 fine-liner 墨线笔触,带有机的、略微不完美的变宽笔触
- **上色**: 极简马克笔风格,只使用极低饱和度的半透明粉彩色(淡橙 + 淡蓝),大面积留白
- **底色**: 暖白奶油色 `#FDF9F4`,干净纯色,不要网格、不要纹理
- **布局**: 大圆角手绘节点容器,内部 padding 充足,呼吸感极强
- **角色**: 右下角一个极简浅灰色轮廓火柴人,配一个单线条手绘椭圆思考气泡,辅助叙事但不抢戏
- **底部金句**: 居中粗体手写中文,底部加荧光笔涂抹效果(淡黄色半透明横条)
- **禁忌**:
  - 不要高饱和配色
  - 不要密集信息堆砌
  - 不要真实照片感
  - 不要深色背景
  - 不要 3D 渲染和光泽效果

## 版式倾向

### 对比类图

- 左右或多栏圆角手绘面板
- 顶部大标题
- 中间用极简箭头或手绘分隔线
- 火柴人可在角落做反应表情

### 流程类图

- 3-5 个圆角步骤容器
- 容器之间用 fine-liner 手绘箭头连接
- 每个步骤内部留足空白
- 底部金句总结流程要义

### 架构类图

- 分层手绘容器,层级清晰
- 层与层之间用细线连接
- 保持极简,不要过度装饰
- 火柴人在角落思考

### 抽象概念类图

- 中心辐射式布局,核心概念居中
- 周围 3-5 个手绘卡片
- 极淡粉彩上色区分不同概念
- 底部荧光笔金句收尾

## 三条自检

1. **是否足够极简**: 画面应该有大量留白,不是每个角落都塞满东西。如果看起来信息太密,就减少节点数量。
2. **配色是否克制**: 只允许极淡的橙色和蓝色粉彩,其他颜色一概不用。如果出现第三种颜色,就错了。
3. **底部金句是否有力**: 荧光笔高亮的总结句必须是读者能记住并复述的话,不是废话总结。
